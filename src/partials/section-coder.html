About this project

How it's coded

AirPortal is built on a Zend MVC framework for the frontend. HTML and CSS are managed with the Foundation library. Other libraries are used to manage JavaScript. While working on AirPortal I've learned how to make the frontend talk to the backend using PHP to connect to the REST and SOAP endpoints.

SaaS

AirPortal is a SaaS that CB Travel Corp provides to its clients. It is primarily a travel management and reporting tool. It brings together custom data provided by the user with data collected from the Global Distribution Network, a source of data for airlines, hotels and other travel bookings. It also combines that data with real time data on travel conditions and events that could pose delays and security threats.

The majority of my coding experience over the last four years has been in various projects within AirPortal. Sometimes it was creating an entirely new view and connecting to new endpoints. Other times it was simply cleaning up the HTML and CSS or converting and existing view to a responsive layout.

A responsive refresh

When I started at Christopherson, AirPortal had no responsive layouts, and the overall design was functional, but not intuitive.

From 2014-2018 I worked with a great team of developers to give AirPortal a complete responsive makeover, while constantly adding new features and refining existing ones.

I've also learned a lot about data architecture and have occasionally made changes directly in the MySQL databases. While it's important for each memeber of the team to know their role, I feel important for frontend developers to understand the data and web services layers to be able ot optimise the user experience and meet deployment deadlines.

About this project

This project is nearly complete. It is currently in staging and the completed site will launch soon.

How it was coded

My go-to framework is Foundation. This site uses Foundation 6, of course with many custom styles. I wrote some custom JavaScript to handle things like the menu animations.

Getting on message

Although the existing Andavo Travel website was an award winning site and was only a few years old, we decided that the message had changed since the site launched and it was time for a complete makeover.

As with most sites, the message was integral to the design. In the previous site, the most prominent feature of the homepage was a slider with a plane, a hotel, and a rental car. These are things that appeal to the traveler, but our target demographic was actually the independent travel agent.

The goal of the site is to entice agents (or independent consultants, as they are known in the industry) to bring their book of clients over to Andavo Travel, and allow us to be their host agency, meaning we would handle vendor contracts, etc.

No stock smiley girls

My vision for this site was to make the site as personal as possible. No smiling girls with headsets. No generic stock photography. I wanted to feature real Andavo people and let them tell their stories.

At the annual AndavoMart convention, a coworker and I set up a studio space to do interviews. We also shot plenty of b-roll footage to help tell the story. I edited the footage into a series of clips like this one.

Knowing that the videos would be a central feature of the website, I designed the site around the look and feel of the videos.

Numbers. Get me numbers

After having a personal feel, my second goal in the design of the site was to present only the most relevant information as concisely as possible. I worked with the marketing department to come up with attention grabbing statistics that could be shown prominently on the homepage.

AndavoVacations.com

AndavoVacations.com is the direct-to-consumer site for Andavo Travel. Users can input their travel interests and be connected to the travel agent that most closely matches their interests.

My contributions: HTML (Foundation 6), CSS (SASS), JavaScript (menu animations, etc.), PHP, MySQL, Lead Designer, Lead Developer more...

Visit http://andavovacations.com

About this project

How it's coded

This site uses Foundation to manage the HTML and CSS, but because it has a data layer it also uses a Zend PHP framework. The travel agents featured on this site create their profiles through AirPortal. That data is saved to a MySQL database and retrieved by the web services. The PHP connects to the REST endpoints and brings the data into the view.

Leveling the playing field

Because all the travel agents are competing for leads, one of the challenges was to make sure that no one was given an unfair advantage when a user comes to the site. To prevent this we randomized the list of agents that appeared in a search.

Increasing adoption

We wanted the advisor profile pages to be something that the travel advisors would want to use. This means that it had be be simple but customizable. A wysywg editor was out of the question because we wanted the pages to have a consistant look, and as a designer, the idea of allowing bold, red, underlined text on a page gives me heart palpitations.

I created several fields that would allow them to make their personal pitch to the user in a clean and consistent way. The suggested fields also made it simpler for the travel advisors to craft their message. I also curated 60 header banners that the advisor could randomize, or choose one that they like best.

The profile pages were so well received that some travel advisors have abandoned their old websites, and now use their advisor profile page as their primary online presence.
